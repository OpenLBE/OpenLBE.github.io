## Mini
The entry point for all tools is Dubnium.Mini.exe, and the first double-click to run will perform an initialization update task. 



## Dubnium.QuickStart
After startup, the testing tools are as follows 

<p align="center">
  <img src="./images/unity/dev-server.png" alt="dev-server" width="60%"/>
</p>

Click the auto button after the ServerHost address to automatically obtain the local testing address.
Click the Start button to start the development test server. The status bar in the bottom right corner displays a clock, indicating that the server has been successfully started.

<p align="center">
  <img src="./images/unity/dev-server-2.png" alt="dev-server-2" width="60%"/>
</p>

To shut down the server, you can click the Stop button 
Please allow when starting redis-server for the first time


<p align="center">
  <img src="./images/unity/dev-server-3.png" alt="dev-server-3" width="60%"/>
</p>

Please allow it when starting Dubnium.ControlServer for the first time; otherwise, the device will not be able to connect to the server.


<p align="center">
  <img src="./images/unity/dev-server-4.png" alt="dev-server-4" width="60%"/>
</p>

The service has been successfully launched, and the test content can be viewed on the Home page 

<p align="center">
  <img src="./images/unity/dev-server-5.png" alt="dev-server-5" width="60%"/>
</p>
